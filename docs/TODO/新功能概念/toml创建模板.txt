以上是安卓项目创建时所需要写入的默认模板

推荐文件名后缀：*.versions.toml

versionCatalogs的内容模板：versionCatalogs { create("{输入的名称文本}") { from(files("{当前创建的toml所在位置}")) } }

在创建时还需要获得当前项目的结构，然后获得工程项目根目录的settings.gradle.kts和build.gradle.kts
然后在这两个文件内部找到 dependencyResolutionManagement，然后取得dependencyResolutionManagement结构内的内容（可能会有需要）
然后在dependencyResolutionManagement中写入创建时的versionCatalogs{ [内容]}
dependencyResolutionManagement {
versionCatalogs { create("{输入的名称文本}") { from(files("{当前创建的toml所在位置}")) } }
}


dependencyResolutionManagement {
[插入的创建时versionCatalogs内容]
[示例的其它属性结构] {
google()
}
}


以上是kotlin dsl，额外的还需要支持groovy dsl
Scope类型有这些：runtimeOnly，implementation，testImplementation，compileOnly，其它Scope
Groovy的类型：
Groovy Long >模板：｛Scope类型｝ group: '{组件空间名}', name: '{组件名称}', version: '{版本内容}'
Groovy Short >模板：{Scope类型} '{内容}' 或者 implementation("{内容}")

规则：
别名首字必须小写：[a-z]([a-zA-Z0-9_.\-])+
